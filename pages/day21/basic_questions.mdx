# Basic Questions — Shortest Paths

Six warm-ups. Trace the algorithms by hand before you implement them — the muscle memory matters more than any single problem solution.

<Steps>

### Question 1 — BFS shortest distance

Given an unweighted, undirected graph with vertices `{A, B, C, D, E, F}` and edges `{A-B, A-C, B-D, C-D, C-E, D-F, E-F}`, what is the shortest distance from `A` to every other vertex?

<ShowHideSolution>
BFS from `A`:

- Layer 0: `{A}`. `dist[A] = 0`.
- Layer 1: `{B, C}`. `dist[B] = dist[C] = 1`.
- Layer 2: `{D, E}`. `dist[D] = dist[E] = 2`.
- Layer 3: `{F}`. `dist[F] = 3`.

Final: `A=0, B=1, C=1, D=2, E=2, F=3`.
</ShowHideSolution>

### Question 2 — Trace Dijkstra

Same graph, but now weighted: `A-B=4, A-C=1, B-D=1, C-D=8, C-E=2, D-F=1, E-F=5`. Find shortest distances from `A`.

<ShowHideSolution>
Heap operations:

- Push `(0, A)`. Pop `(0, A)`. Relax: `dist[B]=4, dist[C]=1`. Push.
- Pop `(1, C)`. Relax: `dist[D] = 1 + 8 = 9, dist[E] = 1 + 2 = 3`. Push.
- Pop `(3, E)`. Relax: `dist[F] = 3 + 5 = 8`. Push.
- Pop `(4, B)`. Relax: `dist[D] = min(9, 4+1) = 5`. Push `(5, D)`.
- Pop `(5, D)`. Relax: `dist[F] = min(8, 5+1) = 6`. Push `(6, F)`.
- Pop `(6, F)`. No outgoing changes.
- Pop `(8, F)` → stale (`8 > dist[F]=6`), skip.
- Pop `(9, D)` → stale, skip.

Final: `A=0, B=4, C=1, D=5, E=3, F=6`.
</ShowHideSolution>

### Question 3 — Why BFS fails on weighted graphs

For the weighted graph in Q2, what answer would naïve BFS give for `dist[D]`, and why is it wrong?

<ShowHideSolution>
BFS treats every edge as weight 1, so it would return the **fewest-edges distance**: `dist[D] = 2` (path `A → C → D`, two edges, or `A → B → D`, two edges).

That's wrong because we care about **total weight**, not edge count. The actual shortest path `A → B → D` has weight `4 + 1 = 5`, and the two-edge path `A → C → D` has weight `1 + 8 = 9`. BFS picks "fewest edges" without regard to weight — Dijkstra is needed when weights differ.
</ShowHideSolution>

### Question 4 — Count Bellman-Ford passes

A directed graph has 5 vertices and 7 edges. Source = vertex 0. After how many passes is Bellman-Ford guaranteed to have finalized all distances (assuming no negative cycle)?

<ShowHideSolution>
`V − 1 = 4` passes.

A shortest path between any two vertices uses at most `V − 1` edges (else it'd repeat a vertex). After pass `k`, every shortest path of edge-length `≤ k` is computed. So 4 passes lock everything in.

In practice it often converges faster — the early-exit "no relaxation this pass → break" optimization is always worth including.
</ShowHideSolution>

### Question 5 — Detecting a negative cycle

A directed graph has edges `{(A, B, 1), (B, C, -3), (C, A, 1)}`. Is there a negative cycle? How would Bellman-Ford detect it?

<ShowHideSolution>
Cycle `A → B → C → A` has total weight `1 + (−3) + 1 = −1 < 0`. **Yes, negative cycle.**

Bellman-Ford detection: after `V − 1 = 2` relaxation passes, do one more pass. If any edge can still be relaxed, there's a negative cycle reachable from the source.

Tracing from source `A`:
- Init: `A=0, B=∞, C=∞`.
- Pass 1: `A→B` relaxes `B=1`, `B→C` relaxes `C=-2`, `C→A` would relax `A = -2 + 1 = -1` — yes, relaxes.
- Pass 2: ripples further. `A=-1`, `B=0`, `C=-3`, and so on.
- After pass 2 (`V-1 = 2`): some state.
- Pass 3 (detection): can still relax. **Negative cycle detected.**
</ShowHideSolution>

### Question 6 — Floyd-Warshall loop order

Why is this Floyd-Warshall implementation **wrong**?

```python
for i in range(n):
    for j in range(n):
        for k in range(n):
            if dist[i][k] + dist[k][j] < dist[i][j]:
                dist[i][j] = dist[i][k] + dist[k][j]
```

<ShowHideSolution>
**The `k` loop must be outermost.** With `i, j, k` in this order, we're computing each `dist[i][j]` by checking all `k`s once — but the `dist[i][k]` and `dist[k][j]` we read may have been updated for other `(i, j)` pairs in earlier iterations with different intermediate sets.

The correct DP is: `dist^(k)[i][j] = min(dist^(k-1)[i][j], dist^(k-1)[i][k] + dist^(k-1)[k][j])`. Collapsing to 2D works **only** when the outer loop is `k`, because then within a single `k`-iteration, the entire matrix is consistently "using intermediates from `{0,...,k}`."

Reorder to `for k: for i: for j:` and it becomes correct.
</ShowHideSolution>
