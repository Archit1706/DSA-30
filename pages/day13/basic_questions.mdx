# Basic Searching Questions

Six warm-up exercises to lock in binary search. Each one is a direct application of the [unified template](/day13/binary_search_template) — by the end you should feel comfortable identifying the monotonic predicate at a glance.

<Steps>

### 1. Find an element in a sorted array

The canonical problem. Use the template — define the predicate `arr[i] >= target` and the answer is "the first true index, if it points at target."

<Tabs items={['C++', 'Python', 'Java']} defaultIndex="0">
    <Tab>
    ```cpp copy
    int binarySearch(const vector<int>& arr, int target) {
        int lo = 0, hi = arr.size();
        while (lo < hi) {
            int mid = lo + (hi - lo) / 2;
            if (arr[mid] >= target) hi = mid;
            else                    lo = mid + 1;
        }
        return (lo < (int)arr.size() && arr[lo] == target) ? lo : -1;
    }
    ```
    </Tab>
    <Tab>
    ```python copy
    def binary_search(arr, target):
        lo, hi = 0, len(arr)
        while lo < hi:
            mid = (lo + hi) // 2
            if arr[mid] >= target: hi = mid
            else:                  lo = mid + 1
        return lo if lo < len(arr) and arr[lo] == target else -1
    ```
    </Tab>
    <Tab>
    ```java copy
    public int binarySearch(int[] arr, int target) {
        int lo = 0, hi = arr.length;
        while (lo < hi) {
            int mid = lo + (hi - lo) / 2;
            if (arr[mid] >= target) hi = mid;
            else                    lo = mid + 1;
        }
        return (lo < arr.length && arr[lo] == target) ? lo : -1;
    }
    ```
    </Tab>
</Tabs>

**Time:** O(log n). **Space:** O(1).

### 2. Find the first occurrence of a duplicated value

Same predicate as classic search — `arr[i] >= target`. The template naturally returns the **leftmost** index because of `hi = mid` (we keep mid as a candidate, narrow leftward whenever possible).

<ShowHideSolution>

```python copy
def first_occurrence(arr, target):
    lo, hi = 0, len(arr)
    while lo < hi:
        mid = (lo + hi) // 2
        if arr[mid] >= target: hi = mid
        else:                  lo = mid + 1
    return lo if lo < len(arr) and arr[lo] == target else -1
```

That's literally identical to problem 1. The template's `hi = mid` (instead of `mid - 1`) is what makes it return the *first* occurrence rather than any occurrence.

For the **last** occurrence, change the predicate to `arr[i] > target` and subtract one from the result:

```python copy
def last_occurrence(arr, target):
    lo, hi = 0, len(arr)
    while lo < hi:
        mid = (lo + hi) // 2
        if arr[mid] > target: hi = mid
        else:                 lo = mid + 1
    pos = lo - 1
    return pos if pos >= 0 and arr[pos] == target else -1
```

Two predicates, one template, both edges of a duplicate run.

</ShowHideSolution>

### 4. Integer Square Root

Find the largest integer `s` such that `s² ≤ x`. Binary search over the **answer space** `[0, x]` — the predicate `s * s > x` is monotonic (false…false…true…true).

<ShowHideSolution>

<Tabs items={['Python', 'C++', 'Java']} defaultIndex="0">
    <Tab>
    ```python copy
    def my_sqrt(x):
        lo, hi = 0, x + 1
        while lo < hi:
            mid = (lo + hi) // 2
            if mid * mid > x: hi = mid
            else:             lo = mid + 1
        return lo - 1
    ```
    </Tab>
    <Tab>
    ```cpp copy
    int mySqrt(int x) {
        long lo = 0, hi = (long)x + 1;
        while (lo < hi) {
            long mid = lo + (hi - lo) / 2;
            if (mid * mid > x) hi = mid;
            else               lo = mid + 1;
        }
        return (int)(lo - 1);
    }
    ```
    </Tab>
    <Tab>
    ```java copy
    public int mySqrt(int x) {
        long lo = 0, hi = (long) x + 1;
        while (lo < hi) {
            long mid = lo + (hi - lo) / 2;
            if (mid * mid > x) hi = mid;
            else               lo = mid + 1;
        }
        return (int)(lo - 1);
    }
    ```
    </Tab>
</Tabs>

This is **binary search on the answer** — the answer space isn't an array, it's the integer range `[0, x]`. The template doesn't care; the predicate does the work.

**Time:** O(log x). **Space:** O(1).

</ShowHideSolution>

### 5. Find a peak element

A "peak" is any element strictly greater than its neighbors. For `arr = [1, 3, 2]`, the peak is `3` at index 1. The trick: binary search by comparing `arr[mid]` with `arr[mid + 1]`.

<ShowHideSolution>

If `arr[mid] < arr[mid + 1]`, a peak must exist to the **right** (the array is rising — keep climbing). If `arr[mid] > arr[mid + 1]`, a peak must exist to the **left or at mid** (the array is falling).

```python copy
def find_peak_element(arr):
    lo, hi = 0, len(arr) - 1
    while lo < hi:
        mid = (lo + hi) // 2
        if arr[mid] < arr[mid + 1]:
            lo = mid + 1
        else:
            hi = mid
    return lo
```

**Time:** O(log n). The predicate "we're past the peak" is monotonic once you frame the problem right — `arr[i] > arr[i+1]` becomes true only after the peak.

</ShowHideSolution>

### 6. Find minimum in a rotated sorted array

A sorted array rotated at some pivot — e.g., `[4, 5, 6, 7, 0, 1, 2]`. Find the minimum.

<ShowHideSolution>

The clever insight: compare `arr[mid]` with `arr[hi]`. If `arr[mid] > arr[hi]`, the minimum is to the **right** of mid. Otherwise, the minimum is to the **left of mid OR at mid itself**.

```python copy
def find_min(arr):
    lo, hi = 0, len(arr) - 1
    while lo < hi:
        mid = (lo + hi) // 2
        if arr[mid] > arr[hi]:
            lo = mid + 1
        else:
            hi = mid
    return arr[lo]
```

The predicate: "is `arr[i]` part of the rotated tail (smaller portion)?" — monotonic when answered relative to `arr[hi]`. First true index = the minimum.

**Time:** O(log n). **Space:** O(1).

</ShowHideSolution>
