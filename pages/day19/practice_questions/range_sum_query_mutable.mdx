### Range Sum Query — Mutable <DifficultyBadge level="medium" />

#### Description

Implement `NumArray`:

- `NumArray(nums)` — initialize with the array.
- `update(index, val)` — set `nums[index] = val`.
- `sumRange(left, right)` — return the sum of `nums[left..right]` inclusive.

#### Examples

```py
> nums = [1, 3, 5]
  sumRange(0, 2)  -> 9
  update(1, 2)               # nums = [1, 2, 5]
  sumRange(0, 2)  -> 8
```

#### Constraints

- `1 <= nums.length <= 3 × 10^4`
- up to `3 × 10^4` calls to `update` + `sumRange`.

This is the textbook "prefix sums won't cut it" problem: with interleaved updates, a prefix-sum array is O(n) per update. We need O(log n) for both.

#### Intuition

A point-update, range-sum query — the exact sweet spot for both a **BIT** (shorter) and a **segment tree** (more general). Below: the BIT, because it's the leanest correct answer. (For the segment-tree version, see [The Template](/day19/the_template).)

#### Code (Fenwick tree)

<Tabs items={['C++', 'Python', 'Java']} defaultIndex="0">
    <Tab>
    ```cpp copy
    class NumArray {
        int n; vector<long long> bit; vector<int> a;
        void add(int i, long long d) { for (i++; i <= n; i += i & (-i)) bit[i] += d; }
        long long pre(int i) { long long s = 0; for (i++; i > 0; i -= i & (-i)) s += bit[i]; return s; }
    public:
        NumArray(vector<int>& nums) : n(nums.size()), bit(n + 1, 0), a(nums) {
            for (int i = 0; i < n; i++) add(i, nums[i]);
        }
        void update(int index, int val) {
            add(index, val - a[index]);          // apply the delta
            a[index] = val;
        }
        int sumRange(int left, int right) {
            return pre(right) - pre(left - 1);
        }
    };
    ```
    </Tab>
    <Tab>
    ```python copy
    class NumArray:
        def __init__(self, nums):
            self.n = len(nums)
            self.bit = [0] * (self.n + 1)
            self.a = [0] * self.n
            for i, v in enumerate(nums):
                self.update(i, v)

        def _add(self, i, d):
            i += 1                                # to 1-indexed
            while i <= self.n:
                self.bit[i] += d
                i += i & (-i)

        def _pre(self, i):
            i += 1
            s = 0
            while i > 0:
                s += self.bit[i]
                i -= i & (-i)
            return s

        def update(self, index, val):
            self._add(index, val - self.a[index])  # delta, not absolute
            self.a[index] = val

        def sumRange(self, left, right):
            return self._pre(right) - self._pre(left - 1)
    ```
    </Tab>
    <Tab>
    ```java copy
    class NumArray {
        int n; long[] bit; int[] a;
        public NumArray(int[] nums) {
            n = nums.length; bit = new long[n + 1]; a = new int[n];
            for (int i = 0; i < n; i++) update(i, nums[i]);
        }
        private void add(int i, long d) { for (i++; i <= n; i += i & (-i)) bit[i] += d; }
        private long pre(int i) { long s = 0; for (i++; i > 0; i -= i & (-i)) s += bit[i]; return s; }
        public void update(int index, int val) {
            add(index, val - a[index]);
            a[index] = val;
        }
        public int sumRange(int left, int right) {
            return (int)(pre(right) - pre(left - 1));
        }
    }
    ```
    </Tab>
</Tabs>

<Callout type="warning">
    **`update` adds the delta, not the value.** A BIT accumulates, so to *set* `nums[index] = val` you push `val − old`, then remember the new value. Adding `val` directly would double-count. Also note the `i++` inside the loops: the BIT is 1-indexed, so 0-based input shifts up by one.
</Callout>

#### Analysis

- **Time:** `update` and `sumRange` are O(log n); construction is O(n log n) (or O(n) with a smarter build).
- **Space:** O(n) for the BIT — versus 4n for a segment tree.
